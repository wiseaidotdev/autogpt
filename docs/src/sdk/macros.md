# The `agents!` Macro and `Auto` Derive

AutoGPT provides two procedural-macro utilities that eliminate boilerplate when working with agents.

## `agents!` Macro

Defined in `autogpt::macros`. Wraps any number of agent values into the `Vec<Arc<Mutex<Box<dyn AgentFunctions>>>>` type expected by `AutoGPT::with`:

```rust
#[macro_export]
macro_rules! agents {
    ( $($agent:expr),* $(,)? ) => {
        vec![
            $(
                std::sync::Arc::new(tokio::sync::Mutex::new(
                    Box::new($agent) as Box<dyn AgentFunctions>
                ))
            ),*
        ]
    };
}
```

### Usage

```rust
use autogpt::prelude::*;

let a = ArchitectGPT::new("A", "Design a system.").await;
let b = BackendGPT::new("B", "Write REST endpoints.", "rust").await;

// Without the macro, verbose:
let agents_vec: Vec<Arc<Mutex<Box<dyn AgentFunctions>>>> = vec![
    Arc::new(Mutex::new(Box::new(a) as Box<dyn AgentFunctions>)),
    Arc::new(Mutex::new(Box::new(b) as Box<dyn AgentFunctions>)),
];

// With the macro, concise:
let agents_vec = agents![a, b];
```

The macro accepts any number of agents and allows a trailing comma.

## `Auto` Derive Macro

Defined in the `auto-derive` crate (re-exported from `autogpt::prelude`). Automatically implements `Agent`, `Functions`, `AsyncFunctions`, and `AgentFunctions` for a struct containing `AgentGPT` and `ClientType` fields.

### Requirements

Your struct **must** contain these two fields with these exact types:

```rust
pub agent: AgentGPT,
pub client: ClientType,
```

Any additional fields are ignored by the macro and are freely available for your custom logic.

### What It Generates

```rust
// These implementations are generated by #[derive(Auto)]:
impl Agent for MyAgent { ... }          // forwards to self.agent
impl Functions for MyAgent { ... }      // returns &self.agent
impl AsyncFunctions for MyAgent { ... } // LLM generate/stream/imagen + LTM
impl AgentFunctions for MyAgent { ... } // blanket supertrait
```

The derived `AsyncFunctions::execute` calls your `Executor::execute` implementation, bridging the trait-object interface with your custom logic.

### Custom Fields in Practice

```rust
use autogpt::prelude::*;

#[derive(Debug, Default, Auto)]
pub struct FileWriterAgent {
    pub agent:      AgentGPT,
    pub client:     ClientType,
    pub output_dir: String,   // custom field, available in execute()
}

#[async_trait]
impl Executor for FileWriterAgent {
    async fn execute<'a>(
        &'a mut self,
        _task: &'a mut Task,
        _execute: bool,
        _browse: bool,
        _max_tries: u64,
    ) -> Result<()> {
        let response = self.generate(self.agent.behavior().as_ref()).await?;
        let path = format!("{}/output.txt", self.output_dir);
        std::fs::write(&path, &response)?;
        println!("Wrote {} bytes to {}", response.len(), path);
        Ok(())
    }
}
```
